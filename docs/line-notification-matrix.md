# LINE 通知對象與額度矩陣

更新日期：2026-06-24

## 控管原則

- 一般公告型通知由試算表 `訂閱者清單` 控管：有填 `LINE_USER_ID` 且未被停用者會收到。
- 若 `訂閱者清單` 有 `是否訂閱`、`是否啟用` 或 `啟用` 欄位，只有啟用值才會收到；若沒有這類欄位，則有 `LINE_USER_ID` 的列視為訂閱者。
- 主管通知由同一張 `訂閱者清單` 的 `是否為主管 = 是` 控管。
- 同仁名單由 `是否為同仁 = 是` 控管，用於每日作業檢核完成率與未填名單，不會直接逐一推播給同仁。
- LINE `reply`、quick reply、rich menu 點擊後的回覆、Loading Animation 不扣訊息額度。
- LINE `push` / `multicast` 主動通知會扣訊息額度，依實際收件人數計算。

## 通知矩陣

| 情境 | 收件對象 | 會收到什麼 | 是否扣 LINE 額度 |
|---|---|---|---|
| 使用者輸入 `狀態` | 發訊者本人 | 填表狀態圖卡 + 每日作業完成率 | 否，reply 不扣 |
| 使用者輸入 `每日作業` | 發訊者本人 | 每日作業檢核填寫入口圖卡 | 否，reply 不扣 |
| 使用者輸入 `異常` | 發訊者本人 | 設備檢查異常清單圖卡 | 否，reply 不扣 |
| 使用者輸入 `通報` | 發訊者本人 | 日常異常事件通報表入口圖卡 | 否，reply 不扣 |
| 使用者輸入 `待處理` | 發訊者本人 | 日常異常事件未結案清單圖卡 | 否，reply 不扣 |
| 使用者輸入 `QR選單` | 發訊者本人 | 日檢/月檢 QR 選單圖卡 | 否，reply 不扣 |
| 未登錄者使用受保護指令 | 發訊者本人 | 登錄提示圖卡、取得我的ID按鈕 | 否，reply 不扣 |
| 設備日檢未填提醒 | 訂閱者清單內可推播者 | 今日未完成檢點提醒圖卡 | 是，主動通知會扣 |
| 設備檢查送出且有異常項 | 訂閱者清單內可推播者 | 設備異常通報圖卡；每個異常項各一則 | 是，主動通知會扣 |
| 需主管簽核的檢查表送出 | `是否為主管 = 是` | 主管待簽核圖卡 | 是，主動通知會扣 |
| 月檢未填提醒 | `是否為主管 = 是` | 本月未完成月檢提醒圖卡與填寫連結文字 | 是，主動通知會扣 |
| 每日作業 16:30 未填提醒 | `是否為主管 = 是` | 尚未填寫每日作業的同仁名單 | 是，主動通知會扣 |
| 每日作業 17:00 未填提醒 | `是否為主管 = 是` | 逾時未完成每日作業的同仁名單 | 是，主動通知會扣 |
| 日常異常事件新通報 | 訂閱者清單內可推播者 | 日常異常事件圖卡 | 是，主動通知會扣 |
| 日常事件狀態為處理中且指定主管 | 指定主管；找不到指定主管時改通知所有主管 | 處理中事件主管意見填寫圖卡 | 是，主動通知會扣 |
| 主管填寫處理中意見 | 訂閱者清單內可推播者 | 主管意見公開圖卡 | 是，主動通知會扣 |
| 日常事件處理完成送審 | 指定主管；找不到指定主管時改通知所有主管 | 日常事件待審圖卡 | 是，主動通知會扣 |
| 主管退回補正 | 訂閱者清單內可推播者 | 退回補正圖卡 | 是，主動通知會扣 |
| 主管同意結案 | 目前不主動推播結案公告 | 只更新表格/PDF/狀態 | 否 |
| Loading Animation | 發訊者的一對一聊天室 | 等待動畫 | 否，不扣 |
