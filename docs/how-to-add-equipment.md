# 新增機具 / 檢查表操作指南

> 適用對象：場地承辦人。**不需要任何程式知識**。

---


## 情況 A：加同類設備（最常見）

**範例**：<位置>已有 1 號天車（CRANE-LJ-001），要再加 2 號天車。

### 步驟

1. 打開 [DB Sheet](https://docs.google.com/spreadsheets/d/1ar4SHX7hqaTiCMjkrPbNXhzR7z8wbcPKifVHshHt83I/edit)
2. 左下角分頁切到「**設備清單**」
3. 在最後一列下面新增：

| 設備代號 | 設備名稱 | 機械編號 | 型式規格 | 設備類別 | 所在位置 | 場地表分頁 | 啟用 |
|---|---|---|---|---|---|---|---|
| `CRANE-LJ-002` | <位置>_天車2號 | 12F36D0130002 | 普通架空移動起重機 | 固定式起重機 | <位置> | 固定式起重機 | TRUE |

**重點**：
- `設備代號` 唯一，自己取（例如 `CRANE-LJ-002`、`CRANE-CH-001`）
- `設備類別` **必須完全一樣**寫「固定式起重機」（系統靠這個對應到日檢/月檢模板）
- `場地表分頁` 寫「固定式起重機」（就是場地表的那個分頁名）
- `啟用` 填 `TRUE`（要關閉就改 `FALSE`，不要直接刪除）

### 完成後

馬上打開 `https://<your-github-username>.github.io/auto-checklist/` 就會看到新設備。
新天車的 QR Code 內容：`https://<your-github-username>.github.io/auto-checklist/daily.html?eqp=CRANE-LJ-002`

## 情況 B：加全新機具種類（堆高機、衝剪機械…）

**範例**：要新增「堆高機」的每日檢點 + 每月檢查。

### 步驟 1：先傳場地表分頁截圖給我看

打開 `115-術科V1` 場地表，把目標分頁（如「內外場-堆高機、移動式、危運、吊籠、一壓」）截圖傳給我。

**為什麼**：如果該分頁結構和「固定式起重機」一樣（每月 2 欄、星期 + 日期 + 內容），你自己加 sheet 就完工。如果不同（例如直式行事曆、或多欄位），我要先改 `Calendar.gs` 加新的解析邏輯。

**判斷標準（你也可以自己看）**：
```
✅ 結構相同：
   列 1：1月 _ 2月 _ ... 12月 _
   列 2：星期 日期 [類別]｜日期 [類別]｜...
   列 3 起：實際資料

❌ 結構不同：（任何不照上面排版的）
```

### 步驟 2：DB「檢查表模板」加兩列

| 表單ID | 設備類別 | 表單名稱 | 週期 | 法規依據 | 填寫規則 | 啟用 |
|---|---|---|---|---|---|---|
| `F-FORK-D` | 堆高機 | 堆高機每日作業前檢點表 | 每日 | 起重升降規則 §128 | 良好「V」/ 無此項「/」/ 不良「X」 | TRUE |
| `F-FORK-M` | 堆高機 | 堆高機每月定期檢查紀錄 | 每月 | 起重升降規則 §99 | 勾選方法、結果、風險、改善 | TRUE |

**重點**：
- `表單ID` 你自己編（建議 `F-` + 類別簡寫 + `D` 或 `M`）
- `設備類別` 跟下面「設備清單」要一致

### 步驟 3：DB「檢查項目」加堆高機的項目

每一項一列。例如堆高機日檢可能是：

| 表單ID | 項目順序 | 項目名稱 | 檢查方法 | 啟用 |
|---|---|---|---|---|
| `F-FORK-D` | 1 | 制動器、離合器、控制裝置 | 目視+操作 | TRUE |
| `F-FORK-D` | 2 | 方向盤、操作裝置 | 操作 | TRUE |
| `F-FORK-D` | 3 | 警報裝置 | 操作 | TRUE |
| `F-FORK-D` | 4 | 前輪 / 後輪 / 連桿 | 目視 | TRUE |
| `F-FORK-D` | 5 | 油壓系統 | 目視 | TRUE |
| `F-FORK-D` | 6 | 升降鏈條、貨叉 | 目視 | TRUE |
| `F-FORK-D` | 7 | 燈光、喇叭 | 操作 | TRUE |
| `F-FORK-M` | 1 | ... |  | TRUE |

（實際項目依法規，可參考勞動部「機械設備檢查表範本」）

### 步驟 4：DB「設備清單」加實體設備

| 設備代號 | 設備名稱 | 機械編號 | 型式規格 | 設備類別 | 所在位置 | 場地表分頁 | 啟用 |
|---|---|---|---|---|---|---|---|
| `FORK-LJ-001` | <位置>堆高機1號 | XXX | 1.5 噸電動堆高機 | **堆高機** | <位置> | **內外場-堆高機、移動式、危運、吊籠、一壓** | TRUE |

**重點**：
- `設備類別` 必須跟「檢查表模板」那邊**完全一樣**
- `場地表分頁` 填場地表中對應的分頁名稱

### 完成

前端會自動列出堆高機，操作員掃 QR 就能填表。**整個過程不用改任何程式、不用重新部署**。

## 常見錯誤

### 1. 新加的設備在前端看不到？
- 檢查「啟用」欄是不是 TRUE（不要寫 true、不要空白）
- 檢查「設備類別」是否與「檢查表模板」拼字完全一致（連空格都不能差）
- 強制重新整理瀏覽器（手機用無痕分頁）

### 2. 提醒信沒寄？
- 檢查場地表那天有沒有資料（沒資料就不寄信）
- 檢查場地表內容有沒有節假日關鍵字（命中就跳過）
- 檢查設備「場地表分頁」欄位是否填正確
- 手動執行 `dailyReminderJob` 看執行記錄找原因

### 3. 簽完沒看到 PDF？
- 開啟 [歸檔資料夾](https://drive.google.com/drive/folders/1Yn4Nj-7qc4TH1aQ7M4NzZCASHO_DQSVz) → 對應設備類別 → 民國年 → 月份
- 也可以在 DB「填報紀錄」最後一列看「PDF連結」欄

### 4. 想看系統現在狀態？
打開：
```
https://script.google.com/macros/s/AKfycbwrAgEXjxRfvQtpGVFqRrHRmr-i5itZZSH9DmrvVl48YWlxFhdxJvEIwQYpR7rv6SkTrQ/exec?api=status
```
會看到所有設定、觸發器、表筆數等。
