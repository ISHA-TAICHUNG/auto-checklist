# 自動檢查表系統 — 使用說明

> 這份文件給三種人看：
> **A. 操作員 / 教師**（每天填表）｜**B. 承辦人**（看異常、回覆提醒信）｜**C. 主管**（看統計）
>
> 不會用到任何電腦程式知識。

---

## A. 操作員 / 教師 — 每天填表

### A-1. 怎麼打開檢查表

**最方便**：掃 QR Code（貼在機具旁邊 / 操作台）

也可以直接打開網址：

| 機具 | 網址 |
|---|---|
| 固定式起重機（天車）| https://<your-github-username>.github.io/auto-checklist/?cat=固定式起重機 |
| 堆高機 | https://<your-github-username>.github.io/auto-checklist/?cat=堆高機 |

打開後，會看到一張「設備卡」清單，每張有「**每日檢點**」和「**每月檢查**」兩顆按鈕。

### A-2. 填寫流程

1. 點「每日檢點」（或月檢）
2. 上方顯示**設備資訊**（不用改）
3. 填**檢查日期**（預設今天）
4. 填**檢點人員姓名**
5. 對每個項目選結果：
   - **固定式起重機日檢**：V（良好）/ /（無此項）/ X（不良）
   - **固定式起重機月檢**：正常 / 異常
   - **堆高機日檢**：○（良好）/ △（尚可）/ X（不良待修）
   - **堆高機月檢**：ˇ（正常）/ X（異常）
6. 選「不良 / 異常」時，下方會跳出：
   - **異常說明欄**：必填，寫清楚是什麼問題
   - **拍照上傳按鈕**：可選，每項最多 4 張，會自動壓縮
7. 滑到最底，在「**檢點人員簽名**」框內手寫簽名
   - 💡 提示：手機**橫向拿**簽名空間更大
   - 寫錯按「清除重簽」
8. 按「**送出檢點表**」
9. 跳「✓ 檢點表已送出」就完成了

### A-3. 注意事項

- 每張表大約 1-2 分鐘可填完
- 簽名是**必填**的（法規要求）
- 異常項一定要填說明 + 建議拍照存證
- 送出後**承辦人會自動收到通知**，不用另外回報
- 同一台機具一天填一次就好（同類別其他機台也算同類別已填）

## B. 承辦人 — 每天會收到提醒、處理異常

### B-1. 你會收到的兩種信

#### 信 1：未填日檢提醒（每天 09:00 寄出）

**寄出條件**：場地表登記今天有用某類機具、但**該類別所有機台都還沒填日檢**

**信件內容**：
```
主旨：[未填日檢提醒] 115/05/20 堆高機

承辦您好，
系統偵測到 115/05/20 堆高機 場地有使用紀錄，
但該類別當日尚未有任何機台完成每日檢點表填報。

機具類別：堆高機
所在位置：<位置>
場地表內容：即2堆機2

請通知當日使用單位 / 操作人員儘速完成檢點
（同類別任一機台填一張即可）：

[前往填寫 →]
```

**你要做的**：通知當天使用單位 / 教師完成填表。

> 💡 同類別只寄 1 封 — 不會 6 台堆高機各寄 1 封

#### 信 2：沒有「提醒寄出」就代表 OK

如果某類機具該日沒登記使用、或同類別已有人填了日檢，**系統不會寄信**。

### B-2. 看待處理異常

**位置**：打開 `自動檢查表-DB`（在「自動檢查表-PDF歸檔」資料夾內）→ 切到「**異常事件**」分頁

每次操作員填表選「X / 異常」就會自動新增一列，欄位包含：

| 欄位 | 說明 |
|---|---|
| 通報日期 / 設備 / 項次 / 項目名稱 | 自動帶入 |
| 異常說明 | 操作員寫的原文 |
| 照片數 | 點 PDF 連結看 |
| **PDF 連結** | 點開看完整檢點表 + 照片 |
| **狀態**（下拉）| 預設「待處理」，處理完改成「已完成」|
| 預計完成日 | 自己填 |
| 實際完成日 | 處理完填 |
| 負責人 | 派工填 |
| 備註 | 寫補充說明 |

### B-3. 承辦人工作流建議

**每天早上 09:00 之後**：
1. 收 email — 看有沒有「未填日檢提醒」→ 通知該單位補填
2. 開 DB「異常事件」分頁
3. 用 Sheets「**篩選**」功能（檢視 → 建立篩選器）→ 狀態 = 待處理
4. 對每筆：
   - 點 PDF 連結看詳情 + 照片
   - 派工 → 填預計完成日、負責人
   - 狀態改「處理中」
5. 處理完 → 狀態改「已完成」、填實際完成日

## C. 主管 — 偶爾看統計

### C-1. 看「這個月有幾件異常」

打開 DB「異常事件」分頁 → 用「樞紐分析表」（資料 → 樞紐分析表）：

範例：「每月每類機具異常統計」
- 列：通報日期（按月分組）
- 欄：設備類別
- 值：項目名稱 計數

### C-2. 看「歸檔 PDF」

打開 Google Drive →「自動檢查表-PDF歸檔」資料夾，結構：

```
固定式起重機/115年/05月/1150519_<設備名稱>_日檢.pdf
                       1150519_<設備名稱>_月檢.pdf
堆高機/115年/05月/1150519_堆高機 A 號_日檢.pdf
                  1150519_堆高機 B 號_日檢.pdf
                  ...
```

點任一 PDF 直接看：含填寫人簽名、所有勾選結果、異常項照片附件頁。

### C-3. 法規查核準備

如果勞檢來查當月填寫紀錄：
1. 打開歸檔資料夾 / 對應月份
2. 全選 PDF → 下載 ZIP
3. 或在 DB「填報紀錄」分頁篩選該月、把「PDF連結」整欄複製給檢查員

## 常見問題

### 我的手機可以用嗎？
- iPhone：iOS 12 以後都可以（包含 iPhone 6s）
- Android：Chrome / Samsung Internet / LINE 內建瀏覽器都可以
- 平板更好用，**現場建議用 iPad / Android 平板**（簽名空間大）

### 加到主畫面，下次點就開
- iPhone Safari：分享 → **加入主畫面**
- Android Chrome：右上選單 → **加到主畫面**

### 簽名怎麼簽比較好？
- 直接用手指或觸控筆
- 手機**橫向拿**空間更大
- 寫錯按「清除重簽」
- 系統會自動裁切空白、置中（不用擔心簽得太小）

### 拍照後想刪掉某張？
照片縮圖右上角有 ✕，點就刪。
- 一張一張拍累積（不是一次選 4 張）
- 一個項目最多 4 張

### 送出後想修改怎麼辦？
**目前不行直接改**。請：
1. 通知承辦人到 DB「填報紀錄」標記為作廢（在備註欄寫「作廢，重填」）
2. 重新填一份新的

### 沒網路可以填嗎？
**不行**。系統需要連 Apps Script API 才能送出。
- WiFi 或 4G/5G 任一連線即可
- 沒網路就先用紙本，事後補填

### 系統壞了 / 看不到設備
1. 強制重整網頁：iPhone Safari 長按重整鈕、Android Chrome 下拉
2. 改用無痕分頁打開
3. 如果還不行，聯絡 [<管理員 email>]

### 提醒信寄到別的信箱怎麼改？
打開 DB「系統設定」分頁 → `reminderEmailTo` 那一格改成新信箱 → 存檔。明天就會寄到新信箱。

### 想換場地表（明年 116 年）
打開 DB「系統設定」分頁 → `venueSheetId` 那一格改成新 Sheet 的 ID。

## 速查表（列印貼旁邊）

| 我要做什麼 | 點哪裡 |
|---|---|
| 填日檢 / 月檢 | 掃機具上的 QR Code → 選機台 → 填表 |
| 看今日待處理異常 | 開 DB → 「異常事件」分頁 → 篩選待處理 |
| 看某張歷史 PDF | 開 Drive →「自動檢查表-PDF歸檔」→ 類別 / 年 / 月 |
| 改提醒收件人 | DB → 「系統設定」→ reminderEmailTo |
| 新增機具 / 設備 | 看 `docs/how-to-add-equipment.md` |

## 系統運作示意（一張圖）

```
操作員手機掃 QR → 填表 → 簽名 → 送出
                                  ↓
                               Apps Script 後端
                            （Google 雲端，自動跑）
                                  ↓
                      ┌───────────┴───────────┐
                      ▼                       ▼
                  PDF 歸檔到 Drive       異常項寫入「異常事件」表
                  （年/月分類）          （狀態=待處理）
                      ↓                       ↓
                  承辦 / 主管隨時可看    承辦處理後改狀態

每天 09:00 自動：
   Apps Script → 讀場地表 → 對未填日檢的類別寄信 → 承辦收信通知補填
```
